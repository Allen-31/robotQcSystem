# 云平台后端接口需求说明（按前端功能反推）

本文档按**一级菜单 → 二级菜单 → 三级菜单**组织，与前端菜单结构一致。每个接口均包含：**1. 接口描述  2. URL  3. 请求方式  4. 请求字段列表  5. 请求参数示例  6. 返回字段列表  7. 返回示例**，且**传参示例与返回示例均完整写出**。

**统一响应外壳**：`{ "code": 200, "msg": "success", "data": { ... }, "timestamp": 1672531200, "traceId": "uuid4" }`。以下返回示例中 `data` 内为业务字段；列表类接口 `data` 可能为 `{ "list": [...], "total": n }` 或直接数组，以各节说明为准。**已联调接口**（登录、用户管理、角色管理、权限管理）保持不变，仅在此按同一格式文档化。

## 一、通用（登录相关）

### 1.1 登录

- **接口描述**：前端登录页提交用户名、密码后调用，验证通过返回 token 与用户信息，用于后续请求鉴权。
- **URL**：`/api/auth/login`
- **请求方式**：POST

**请求字段**

| 参数名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| username | Y | string | 用户名（登录账号） |
| password | Y | string | 密码 |
| remember | N | boolean | 是否记住登录状态 |

**请求参数示例**

```json
{
    "username": "admin",
    "password": "123456",
    "remember": true
}
```

**返回字段**

| 参数名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| token | Y | string | 访问令牌，请求头携带 |
| user | Y | object | 用户信息 |
| user.code | Y | string | 用户编码 |
| user.displayName | Y | string | 显示名称 |
| user.roles | Y | array | 角色编码列表 |

**返回示例**

```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "user": {
            "code": "admin",
            "displayName": "系统管理员",
            "roles": ["admin"]
        }
    },
    "timestamp": 1672531200,
    "traceId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

#### 2.1.3 工单管理（三级菜单）

**接口 2.1.3.1 工单列表（分页/筛选）**

- **接口描述**：质检业务-工单管理页获取工单列表。**工单管理页仅展示未执行、执行中、已暂停的工单**（即 `status` 为 `pending` / `running` / `paused`）；其他场景（如质检台历史工单、当前工单）通过查询参数控制。同一接口通过不同查询条件支持：
  - **工单管理页列表**：不传 status 或传 `status=pending,running,paused`，仅返回上述三种状态的工单。
  - **根据质检台查历史工单列表**：传 `stationCode` 为质检台编号，可配合 dateFrom/dateTo、status 等筛选。
  - **根据质检台查当前工单**：传 `stationCode` + `status=running`（质检中），并 `pageSize=1`，按开始时间或创建时间倒序取**最新一条**，即当前工单；若 list 为空则表示该质检台无进行中工单。
- **URL**：`/api/qc/work-orders`
- **请求方式**：GET

**请求字段**（查询参数，拼在 URL）

| 参数名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| page | N | number | 页码，默认 1 |
| pageSize | N | number | 每页条数，默认 20；查当前工单时传 1 |
| keyword | N | string | 工单号/线束编码等 |
| status | N | string | pending / running（执行中）/ paused（已暂停）/ finished / ng / cancelled；工单管理页仅查 pending、running、paused |
| qualityResult | N | string | ok / ng / pending |
| stationCode | N | string | 工位/质检台编码，传则只返回该质检台下的工单 |
| harnessType | N | string | 线束类型 |
| dateFrom | N | string | 创建开始日期，如 2026-03-01 |
| dateTo | N | string | 创建结束日期，如 2026-03-06 |
| orderBy | N | string | 排序，如 startedAt_desc / createdAt_desc；查当前工单时建议按开始时间倒序取最新一条 |

**请求参数示例**

```
# 常规分页
GET /api/qc/work-orders?page=1&pageSize=20&status=running&dateFrom=2026-03-01&dateTo=2026-03-06

# 根据质检台查历史工单列表（如质检台管理页「历史工单」）
GET /api/qc/work-orders?page=1&pageSize=20&stationCode=ST-001&dateFrom=2026-03-01&dateTo=2026-03-06

# 根据质检台查当前工单：质检台 + 状态=质检中 + 取最新一条（list[0] 即当前工单，空则无）
GET /api/qc/work-orders?stationCode=ST-001&status=running&pageSize=1&orderBy=startedAt_desc
```

**返回字段**

| 参数名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| list | Y | array | 工单列表 |
| list[].id | Y | string | 工单ID |
| list[].workOrderNo | Y | string | 工单号 |
| list[].harnessCode | Y | string | 线束编码 |
| list[].movingDuration | Y | number | 搬运耗时 |
| list[].harnessType | Y | string | 线束类型 |
| list[].stationCode | Y | string | 工位编码 |
| list[].status | Y | string | pending / running / paused / finished / ng / cancelled |
| list[].qualityResult | Y | string | ok / ng / pending |
| list[].taskIds | Y | array | 关联调度ID列表（前端可展示为首条为调度编码） |
| list[].detectionDuration | Y | number | 检测耗时 |
| list[].createdAt | Y | string | 创建时间 |
| list[].startedAt | N | string | 开始时间 |
| list[].endedAt | N | string | 结束时间 |
| list[].defectType | N | string | 缺陷类型 |
| list[].defectDescription | N | string | 缺陷描述 |
| total | Y | number | 总条数 |

**返回示例**

```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "list": [
            {
                "id": "WO-1",
                "workOrderNo": "WO-20260228-201",
                "harnessCode": "HB-20260228-201",
                "movingDuration": 12.4,

                "stationCode": "A1-01",

                "endedAt": "-",
                "defectType": "-",
                "defectDescription": "-"
            }
        ],
        "total": 1
    },
```

- **接口描述**：质检区管理或统计页中，根据**某个质检区**（工作站）及**时间维度（日/周）**查询质检相关汇总指标，用于展示「今日」或「本周」的质检总数、平均用时、质检失败量等卡片数据。
- **URL**：`/api/qc/statistics/summary` 或 `/api/qc/statistics/indicators`

|  |  |  | 不传时：day 默认当天，week 默认当周 |

**请求参数示例**

```
# 查询某质检区「今日」指标
GET /api/qc/statistics/summary?workstationId=WS-CFG-001&dimension=day&date=2026-03-06

# 查询某质检区「本周」指标
GET /api/qc/statistics/summary?workstationId=WS-CFG-001&dimension=week&date=2026-03-06

GET /api/qc/statistics/summary?dimension=day
```

**返回字段**

| 参数名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| workstationId | N | string | 质检区ID/编码，全厂汇总时可为空 |
| dimension | Y | string | day / week |
| date | Y | string | 基准日期 |
| periodLabel | N | string | 前端展示用，如「今日」「本周」或「2026-03-06」「2026-03-04 ~ 2026-03-10」 |
| totalCount | Y | number | 质检总数 |
| avgDurationMinutes | Y | number | 平均用时（分钟） |
| failCount | Y | number | 质检失败量（NG 数） |
| defectCount | N | number | 缺陷数 |
| reinspectionCount | N | number | 复检数 |

**返回示例**

```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "workstationId": "WS-CFG-001",
        "workstationName": "总装一线工作站",
        "dimension": "day",
        "date": "2026-03-06",
        "periodLabel": "今日",
        "totalCount": 811,
```

**说明**：若后端与 2.3.1.1 共用统计逻辑，可由同一服务在 dimension=day 时按单日聚合、dimension=week 时按周聚合后返回上述结构；前端「今日/本周」切换时传不同 dimension 与 date 即可。
